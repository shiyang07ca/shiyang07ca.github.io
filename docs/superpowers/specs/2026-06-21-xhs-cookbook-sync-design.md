# 小红书收藏同步到菜谱设计

## 目标

将已登录 Edge 中的小红书收藏自动收集为可核验的来源清单，把图文和视频整理为本站厨房备忘录的菜谱，并且只在同步成功得到确认后取消对应收藏。

## 范围与边界

- 采集器只在用户已登录的小红书页面内运行，只读取页面可见内容。
- 采集结果不包含 Cookie、登录令牌、请求头或其他账号凭据。
- 视频笔记优先读取页面可见字幕；没有字幕时记录视频来源和待转写状态。视频文件转写只能使用用户合法提供或明确授权下载的文件。
- 菜谱同步、站点构建核验和取消收藏是三个独立阶段。取消收藏不会作为同步脚本的副作用执行。
- 只修改与菜谱同步直接相关的文件，不调整既有分类文件的组织方式。

## 数据流

```text
收藏页自动滚动
  -> 来源清单 JSON
  -> 笔记正文与视频转写记录
  -> 菜谱 Markdown
  -> 严格站点构建与来源核验
  -> 成功清单 JSON
  -> 用户确认
  -> 仅取消成功清单中的收藏
```

## 采集阶段

在 Edge 开发者工具的 Snippets 中运行单文件脚本。脚本持续滚动收藏页，直到连续多轮滚动不再发现新的笔记链接。每条笔记输出以下字段：

- `source_url`：规范化后的笔记链接，作为去重键。
- `title`：卡片可见标题；缺失时为空字符串。
- `author`：卡片可见作者；缺失时为空字符串。
- `media_type`：`article`、`video` 或 `unknown`。
- `collected_at`：采集时的 ISO 8601 时间。

脚本在浏览器下载一个 JSON 文件。若页面因登录失效、加载错误或滚动容器无法定位，脚本停止并明确输出错误，不生成可用于取消收藏的成功清单。

采集器不取消收藏、不修改页面内容、不发送采集数据到第三方服务。

## 导入阶段

导入程序读取来源清单，并为每条笔记维护明确状态：`pending`、`imported`、`needs_review` 或 `failed`。

- 图文笔记从正文中提取食材、调料和做法。
- 视频笔记从字幕或转写文本中提取相同字段；无法确认的用量、火候和步骤标为“待复核”，不补造数据。
- 菜谱写入已有分类文件，保持一个分类文件一个一级标题、每道菜一个二级标题、食材和做法分别使用三级标题的统一排版。
- 每道导入菜谱保存原始来源链接，避免重复导入，也使后续清理可以追溯。

只有成功写入菜谱且来源链接能被再次找到的条目才标记为 `imported`。视频没有可用转写时保持 `needs_review`，不会视作同步成功。

## 核验与成功清单

导入结束后执行 `rtk mkdocs build --strict`。随后生成成功清单，其中仅包含：

- 状态为 `imported`；
- 已写入菜谱文件；
- 来源链接与来源清单完全匹配；
- 没有重复来源链接。

汇总报告必须列出来源总数、成功数、待复核数、失败数与跳过数。除非用户明确指定排除项，否则存在待复核或失败条目时不得生成可执行的取消收藏清单。

## 取消收藏阶段

取消收藏脚本独立于采集和导入脚本。运行前必须展示本次将处理的来源数量和链接清单，并在操作当时再次获得用户确认。

脚本只对成功清单中的来源执行取消收藏。每次操作后验证页面收藏状态；失败项目保留在清单中，不影响其他已验证项目。操作结果写入审计记录，包含来源链接、执行时间与成功或失败状态。

## 错误处理与恢复

- 采集过程可以重复运行，按照 `source_url` 去重。
- 导入过程可以从 `pending`、`needs_review` 和 `failed` 状态继续，不重复写入已导入菜谱。
- 取消收藏过程只读取不可变的成功清单，不根据实时页面猜测同步状态。
- 任意阶段出错时保留中间文件，便于重跑和人工核验。

## 验证

- 使用脱敏的固定 HTML 片段测试自动滚动终止、链接去重和 JSON 输出结构。
- 使用固定来源清单测试分类映射、重复链接跳过和待复核标记。
- 对导入结果执行 `rtk mkdocs build --strict`。
- 使用模拟收藏状态测试取消收藏脚本只接收成功清单中的链接，并逐条记录结果。
